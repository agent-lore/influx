## Codebase Patterns
- Use `uv run` prefix for all Python tooling (pytest, ruff, pyright)
- Error hierarchy lives in `src/influx/errors.py` — LCMAError, LithosError, ConfigError all derive from InfluxError
- LCMA wrappers are async methods on `LithosClient`: `retrieve()`, `edge_upsert()`, `task_create()`, `task_complete()`
- `_call_lcma_tool()` helper translates `McpError` and `isError` responses into `LCMAError("unknown_tool")`
- FastMCP tool handlers cannot use `**kwargs` — use explicit named parameters
- Config thresholds including `lcma_edge_score` live in `ProfileThresholds` in `src/influx/config.py`
- Tier3 `builds_on` items and Tier1 `contributions` are defined in `src/influx/schemas.py` (Tier3Extraction, Tier1Enrichment)
- Tests organized under `tests/unit/`, `tests/contract/`, `tests/integration/`
- Project uses `from __future__ import annotations` consistently
- `FakeLithosServer` in `tests/contract/test_lithos_client.py` has all LCMA tools (retrieve, edge_upsert, task_create, task_complete) with FIFO response queues — use it for integration tests
- `_NoopClient` stubs in unit tests that call `run_profile` with non-backfill RunKind must provide `task_create()` and `task_complete()` methods that return valid `CallToolResult`
- `run_task_id` is available in `run_profile()` scope after task_create — downstream stories (US-005, US-006) can access it for after_write hooks
- `after_write()` in `lcma.py` takes `client`, `title`, `contributions`, `run_task_id`, `profile`, `lcma_edge_score` — returns `list[dict]` of `{"title", "score"}` for webhook digest
- Retrieve response shape: `{"results": [{"title": ..., "score": ..., "receipt_id": ..., "note_id": ...}]}`
- `HighlightItem` already has `related_in_lithos: list[dict[str, Any]]` with `default_factory=list` — attach after_write results here
- Item provider items can carry `contributions: list[str]` for Tier 1 contributions; `item.get("contributions")` returns None if absent
- Item provider items can carry `builds_on: list[str]` for Tier 3 builds_on items; `item.get("builds_on")` returns None if absent
- `resolve_builds_on()` in `lcma.py` takes `client` and `builds_on` — uses `cache_lookup` + exact `source_url` match to upsert `builds_on` edges
- `cache_lookup` hit response shape: `{"hit": true, "source_url": "...", "note_id": "...", "title": "..."}` — verify `source_url` matches before upserting edge
