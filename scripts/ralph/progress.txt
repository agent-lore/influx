## Codebase Patterns
- Use `asyncio.Lock` per profile for coordinator ‚Äî safe on single event loop, no race between check-and-acquire
- Dev dependencies (ruff, pyright, pytest-asyncio, respx) need explicit pip install in fresh envs ‚Äî not auto-installed from dependency-groups
- Ruff enforces import sorting (I001) and combined `with` statements (SIM117) ‚Äî run `ruff check --fix` after creating new files
- Test fixture `influx_config_env` in conftest.py provides realistic v0.7 TOML + TEST_PROVIDER_API_KEY env var
- pytest-asyncio configured with `asyncio_mode = "auto"` ‚Äî no need for `@pytest.mark.asyncio` decorators (but they don't hurt)
- `ProbeLoop(config, interval=30.0)` is the probe entry point; `.run_once()` for sync, `.start()`/`.stop()` for async background
- `ProbeLoop.state` returns `ProbeState` with `.is_ready`, `.overall_status`, `.lithos`, `.llm_credentials`
- `load_config(check_api_keys=False)` skips provider key validation ‚Äî use for `serve` mode where probes handle it
- `_make_config(providers=...)` helper in test_probes.py builds minimal AppConfig for unit tests without TOML files
- `InfluxScheduler(config, coordinator)` wraps APScheduler; `.start()` registers jobs, `.stop()` shuts down; `._fire_profile()` acquires lock + calls `run_profile`
- `run_profile(profile, kind, run_range)` is the stub seam ‚Äî PRDs 04/06/09 replace the body
- In `_make_config` helpers, use `if x is not None` instead of `x or default` ‚Äî empty list is falsy in Python
- APScheduler 3.x `AsyncIOScheduler` needs a running event loop ‚Äî make registration tests async
