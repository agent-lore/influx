## Codebase Patterns
- Use `asyncio.Lock` per profile for coordinator тАФ safe on single event loop, no race between check-and-acquire
- Dev dependencies (ruff, pyright, pytest-asyncio, respx) need explicit pip install in fresh envs тАФ not auto-installed from dependency-groups
- Ruff enforces import sorting (I001) and combined `with` statements (SIM117) тАФ run `ruff check --fix` after creating new files
- Test fixture `influx_config_env` in conftest.py provides realistic v0.7 TOML + TEST_PROVIDER_API_KEY env var
- pytest-asyncio configured with `asyncio_mode = "auto"` тАФ no need for `@pytest.mark.asyncio` decorators (but they don't hurt)
- `ProbeLoop(config, interval=30.0)` is the probe entry point; `.run_once()` for sync, `.start()`/`.stop()` for async background
- `ProbeLoop.state` returns `ProbeState` with `.is_ready`, `.overall_status`, `.lithos`, `.llm_credentials`
- `load_config(check_api_keys=False)` skips provider key validation тАФ use for `serve` mode where probes handle it
- `_make_config(providers=...)` helper in test_probes.py builds minimal AppConfig for unit tests without TOML files
- `InfluxScheduler(config, coordinator)` wraps APScheduler; `.start()` registers jobs, `.stop()` shuts down; `._fire_profile()` acquires lock + calls `run_profile`
- `run_profile(profile, kind, run_range)` is the stub seam тАФ PRDs 04/06/09 replace the body
- In `_make_config` helpers, use `if x is not None` instead of `x or default` тАФ empty list is falsy in Python
- APScheduler 3.x `AsyncIOScheduler` needs a running event loop тАФ make registration tests async
- HTTP API router in `src/influx/http_api.py` тАФ dependencies stored on `app.state` (config, coordinator, scheduler, probe_loop)
- FastAPI TestClient (sync) works for most tests; scheduler start requires `async def` test (needs event loop)
- Ruff UP017: use `datetime.UTC` alias instead of `datetime.timezone.utc`
- POST /runs uses Pydantic `model_validator(mode="after")` for exactly-one-of validation тАФ FastAPI auto-returns 422 on failure
- `BackfillRequest` uses Pydantic `Field(alias="from")` for reserved-word JSON keys; needs `populate_by_name=True` in `model_config`
- `_backfill_estimate_override` module-level var in `http_api.py` is the test hook for forcing estimator values тАФ use `monkeypatch.setattr`
- Background task via `asyncio.get_event_loop().create_task()` тАФ acquires lock before responding 202, releases in background after `run_profile`
- To test 409 overlap with sync TestClient, pre-acquire the lock via `asyncio.new_event_loop()` or use `async def` test
- `InfluxService(config)` is the top-level service; `.start()` starts probe loop + scheduler, `.stop()` shuts down cleanly
- `create_app(config)` builds FastAPI app with all deps on `app.state` without starting background tasks
- `validate_bind_host(host, allow_remote_admin=...)` uses `ipaddress.ip_address().is_loopback` for the bind guard (AC-03-D)
- `resolve_bind_address()` reads `INFLUX_ADMIN_BIND_HOST` (default `127.0.0.1`) and `INFLUX_ADMIN_PORT` (default `8080`)
- `SecurityConfig.allow_remote_admin` (default False) and `ScheduleConfig.shutdown_grace_seconds` (default 30) are the US-007 schema extensions
- `InfluxService(config, with_lifespan=True)` wires the lifespan context manager into the FastAPI app for uvicorn-driven startup/shutdown
- `uvicorn.run(app, timeout_graceful_shutdown=N)` bounds shutdown wait; uvicorn 0.46+ exits -15 on SIGTERM (clean shutdown but signal-terminated return code), exit 0 on SIGINT
- To monkeypatch locally-imported `uvicorn.run()`, import uvicorn at test scope and use `monkeypatch.setattr(uvicorn, "run", fake)` тАФ not `"module.uvicorn.run"`
- Subprocess serve tests: use `_find_free_port()` via `socket.bind(("127.0.0.1", 0))` and `_wait_for_live()` polling `/live` with httpx
- CLI HTTP client handlers use `httpx.post()` (sync) with `timeout=10.0`; `httpx.ConnectError` maps to exit 2, `409` maps to exit 1
- Use `respx.mock` decorator + `respx.post(...).mock(return_value=...)` for unit-testing CLI HTTP clients without a running server
- `_admin_base_url()` reads `INFLUX_ADMIN_PORT` from env, targets `127.0.0.1` тАФ single source of truth for CLIтЖТservice URL
