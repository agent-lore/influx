## Codebase Patterns
- Rejection-rate module: `src/influx/rejection_rate.py` — in-memory run counter + tag store; `reset()` for test isolation
- `record_filter_result()` is called per item in `_run_profile_body()` — records tags from provider output
- `on_run_complete()` emits AC-M4-4 stats every run + rejection-rate log at cadence boundary
- `fetch_rejection_titles()` in `feedback.py` queries Lithos for `influx:rejected:<profile>` tags — reused by rejection_rate module
- OTEL package imports: use `opentelemetry.sdk.trace` and `opentelemetry.trace` (not `opentelemetry.api` or `opentelemetry.sdk`)
- No `InMemorySpanExporter` in the installed OTEL SDK version — use a custom `_CollectingExporter(SpanExporter)` for test span capture
- Config already has `TelemetryConfig` model and env overrides for `INFLUX_OTEL_ENABLED` and `INFLUX_OTEL_CONSOLE_FALLBACK` in `config.py`
- Test fixtures: use `influx_config_env` from `tests/conftest.py` for config-based tests; `fake_lithos_sse_url` for integration tests needing a reachable Lithos endpoint
- Import-stubbing pattern for simulating absent packages: save `sys.modules` keys, monkeypatch-delete them, override `builtins.__import__`, restore in finally block
- Service-level tests: use `FastAPI` + `TestClient` for boot/serve assertions; `run_profile()` for run lifecycle
- Telemetry singleton: use `get_tracer(force_rebuild=True)` in tests to pick up env changes
- OTEL global TracerProvider: do NOT use `trace.set_tracer_provider()` — use `provider.get_tracer()` from a local `TracerProvider` to avoid "Overriding not allowed" warnings
- Console fallback test pattern: use `ConsoleSpanExporter(out=io.StringIO())` instead of capsys/capfd
- `_make_collecting_tracer()` helper in test_telemetry.py: returns `(InfluxTracer, collected_spans_list)` for span assertions
- `run_profile()` delegates to `_run_profile_body()` — use `patch("influx.scheduler._run_profile_body")` to test span wiring without Lithos
- `current_run_id` ContextVar in `telemetry.py` — set by `run_profile()`, read by downstream span sites (filter, fetchers) for `influx.run_id` attribute
- `ArxivItem` fields: `arxiv_id`, `title`, `abstract`, `published` (datetime), `categories` — no `authors`, `updated`, `pdf_url`
- `fetch_arxiv` is synchronous — mock with `MagicMock`, not `AsyncMock`
- Fetch/storage spans belong at the provider level (inside `make_*_item_provider`) where `profile` is available, not in raw fetch functions
- Enrichment tier spans belong in `build_arxiv_note_item` where `profile_name` is available — `influx.item_count` is 1 for per-item enrichment
- `Tier1Enrichment` fields: `contributions` (list[str]), `method`, `result`, `relevance` — no `summary`/`keywords`
- `Tier3Extraction` fields: `claims` (list[str] min 1), `datasets`, `builds_on`, `open_questions`, `potential_connections` — no `methodology`/`key_findings`/`limitations`
