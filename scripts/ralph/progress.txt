## Codebase Patterns
- Use `from __future__ import annotations` at top of every module
- Pydantic v2 models in `src/influx/` follow `BaseModel` with `Field(default_factory=...)` pattern
- Tests use class-based grouping (`class TestXxx:`) with descriptive method names
- Use `.venv/bin/python -m pytest` to run tests (uv-managed venv)
- Use `.venv/bin/python -m pyright` for typechecks, `.venv/bin/python -m ruff check` for lint
- ruff line length limit is 88 chars (default); split long strings with implicit concatenation
- `src/influx/errors.py` defines the exception hierarchy; all custom exceptions derive from `InfluxError`
- `src/influx/slugs.py` provides FR-ST-2 slug validation — reuse it, don't re-implement
- Git identity: `git config user.email "ralph@influx.dev" && git config user.name "Ralph Agent"`
- `src/influx/notes.py` owns note parse/render; `parse_note()` returns `ParsedNote(frontmatter_raw, title, sections, user_notes)`
- Canonical note format: `---\n<yaml>\n---\n# Title\n\n## Archive\n...\n## Summary\n...\n## User Notes\n...`
- User Notes region includes the `## User Notes` heading line itself and everything to EOF
- `merge_tags(existing_tags=..., new_tags=...)` handles Influx-owned replacement, profile union with rejection guard, external preservation
- `recompute_confidence(existing_confidence=..., current_max_score=...)` returns `max(existing, score/10.0)`
- Influx-owned prefixes: `source:`, `arxiv-id:`, `cat:`, `text:`, `ingested-by:`, `schema:`; exact: `full-text`, `influx:repair-needed`, `influx:archive-missing`, `influx:deep-extracted`, `influx:text-terminal`
- `render_archive_section(path)` renders `## Archive` section; `parse_archive_path(note)` extracts path from parsed note
- `validate_archive_tag_invariant(archive_path=, tags=)` enforces path + archive-missing mutual exclusion
- Archive path lives ONLY in `## Archive` section body — never in frontmatter or tags
- `render_note(title=, source_url=, tags=, confidence=, archive_path=, summary=, keywords=, profile_entries=, user_notes=)` renders full canonical note
- `ProfileRelevanceEntry(profile_name, score, reason)` is the data class for profile relevance entries
- `parse_profile_relevance(parsed_note)` extracts `ProfileRelevanceEntry` list from `## Profile Relevance` section
- `build_profile_relevance_for_rewrite(old_entries=, new_entries=, tags=)` merges profile entries, keeping old for rejected profiles
- Frontmatter rendered manually (no PyYAML dep): note_type, namespace, source_url, tags, confidence
- Golden file fixture at `tests/fixtures/golden_note.md` — update it when canonical format changes
- `storage.py` owns archive path construction and download; `build_archive_path(...)` returns `(Path, str)` tuple; `download_archive(...)` returns `ArchiveResult(ok, rel_posix_path, error)`
- `ArchivePathError` derives from `InfluxError` — raised for unsafe IDs (traversal, absolute paths) and invalid source slugs
- Path safety: `_reject_unsafe_id()` checks for `..` components and absolute paths BEFORE `resolve().is_relative_to()` as a defense-in-depth measure
- `download_archive(url=, archive_root=, source=, item_id=, ...)` uses `guarded_fetch` from `http_client.py` for SSRF/oversize/timeout protection
- `ArchiveResult(ok=bool, rel_posix_path=str|None, error=str)` — on failure, caller renders empty `## Archive` body + `influx:repair-needed` + `influx:archive-missing` tags
- `NetworkError` from `errors.py` carries `kind` field (`ssrf`, `oversize`, `timeout`, `network`) — use it for structured failure handling
