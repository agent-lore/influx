## Codebase Patterns
- Use `feedparser.parse(content)` for RSS/Atom parsing; access entry fields via `str(entry.get("field", ""))` to satisfy pyright
- `RssSourceEntry` has Literal type for `source_tag` — test helpers must use `Literal["rss", "blog"]` not bare `str`
- Frozen dataclasses with `slots=True` for immutable records (matches arXiv pattern)
- Test fixtures live under `tests/fixtures/<source>/*.xml`
- Config schema already has `RssSourceEntry` and `ProfileSources.rss` from prior PRDs
- `slugify_feed_name()` in `src/influx/slugs.py` for FR-ST-2 slug generation
- `url_hash(source_url)` in `src/influx/urls.py` returns 10-char hex SHA-256 of normalised URL for archive filename disambiguation
- RSS archive `item_id` for `build_archive_path()` = `f"{feed_slug}-{YYYY-MM-DD}-{url_hash}"`
- `download_archive()` in `storage.py` accepts `expected_content_type` param (default `"pdf"`, pass `"html"` for RSS)
- `build_rss_note_item()` in `sources/rss.py` builds complete `ProfileItem` dict with archive download + article extraction (fallback to feed summary) + note rendering
- RSS note path = `articles/{source_tag}/{YYYY}/{MM}`, tags include `source:{source_tag}`, archive under `{source_tag}/{YYYY}/{MM}/`
- Integration tests for RSS use `FakeArticleServer` (simple `http.server` serving HTML) with `SecurityConfig(allow_private_ips=True)`
- Rejection authority enforced at 3 layers: `merge_tags()`, `merge_profile_relevance_union()`, and `build_negative_examples_block()` — all respect `influx:rejected:<profile>` tags
- To capture filter_prompt in tests, use a capturing item_provider that records `filter_prompt` per profile
- `estimate_backfill_items(profile, config, *, days=, date_from=, date_to=)` — real naive estimator using `days × categories × max_results`; test overrides via `_backfill_estimate_override` (monkeypatch) or `INFLUX_TEST_BACKFILL_ESTIMATE` (env var)
- Tests that POST `/backfills` with default config must pass `"confirm": True` unless testing the confirm gate itself (default config yields >1000 items)
- `run_backfill()` in `backfill.py` is the canonical entry point for backfill runs — wraps `run_profile(kind=BACKFILL)` with proper `run_range`
- FR-BF-2: backfill cache-hit skip is in `run_profile`'s cache-hit branch — `if kind == RunKind.BACKFILL: continue` before the write attempt
- CLI subprocess tests need `--confirm` flag (not `INFLUX_TEST_BACKFILL_ESTIMATE` env var) because the estimator runs in the server process, not the CLI subprocess
- All run kinds (scheduled, manual, backfill) create `lithos_task_create` tasks; tag is `influx:backfill` for backfill, `influx:run` otherwise — `_NoopClient` mocks in scheduler tests must include `task_create`/`task_complete`
